kernel.shell.site

Site control shared both by clients and the server.

Generates and receives messages at a site. Modifies the global context. Site control operations are assumed to run atomically in a single thread, no synchronization is performed.

GC!

next-FM

(next-FM MCGS CDAG HB base-FM)

Based on the result the MOVIC algorithm returned, checks whether one CG was produced, in that case applying and returning the correct feature model. Otherwise, returns :conflict.

receive-message!

(receive-message! message)

Receives a message at a site. If a server vector clock is attached, updates its garbage collector (irrelevant for mesh topologies). Checks which kind of message has been received and dispatches to the according receive function. Returns the (updated) current feature model.

view source

receive-operation!

(receive-operation! message)

Receives an operation message at a site. Extracts the operation from the message, then updating the global context. Calls the MOVIC algorithm with the new operation after the CDAG, HB and CC have been updated. Returns the (updated) current feature model.
